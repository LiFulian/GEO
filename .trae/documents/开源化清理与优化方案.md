项目 `/Users/lfl/Documents/软件开发/geo` 已具备完整开源基线（README/CHANGELOG/SECURITY/CoC/CI/Dependabot/Issue+PR 模板/Docker/双远程仓库 GitHub+Gitee），服务器 DEMO（geo.fulianli.top）已稳定运行 5h+ 并通过 8/8 合规检查。但当前存在三类问题阻碍正式开源发布：

## 实施步骤

### Step 1: 创建 .local/ 隔离个人文件

将个人文件移入 `.local/`（会被 gitignore），保持仓库根目录干净：

- `mkdir -p .local/`
- `mv SERVER_DEPLOY_GUIDE.md .local/`
- `mv SERVER_PROJECTS.md .local/`
- `mv opencode.json .local/`（OpenCode + PocketBase MCP 个人配置）

### Step 2: 测试文件移到 tests/

CI 依赖 `test_*.cjs`（参见 [.github/workflows/ci.yml:49](file:///Users/lfl/Documents/软件开发/geo/.github/workflows/ci.yml)），不能删，移到 `tests/` 目录：

- `mkdir -p tests/`
- `mv test_article_draft.cjs test_fixes_smoke.cjs test_pdetail_save.cjs test_ux_fixes.cjs tests/`
- 更新 [.github/workflows/ci.yml](file:///Users/lfl/Documents/软件开发/geo/.github/workflows/ci.yml) 第 49 行：`for t in ../test_*.cjs` → `for t in ../tests/test_*.cjs`
- 更新 [.github/PULL_REQUEST_TEMPLATE.md](file:///Users/lfl/Documents/软件开发/geo/.github/PULL_REQUEST_TEMPLATE.md) 第 28 行：`node test_*.cjs` → `node tests/test_*.cjs`

### Step 3: 删除冗余临时文件

- `rm geo.fulianli.top_nginx.zip`（部署已完成，服务器配置在 nginx 站点文件里）
- `find . -name .DS_Store -delete`（macOS 垃圾文件，所有目录）

### Step 4: 完善 .gitignore

在现有 [.gitignore](file:///Users/lfl/Documents/软件开发/geo/.gitignore) 基础上追加：

```gitignore
# 个人文件（本地保留，不进仓库）
.local/
.trae/
.workbuddy/
opencode.json
SERVER_DEPLOY_GUIDE.md
SERVER_PROJECTS.md

# macOS
.DS_Store

# 临时压缩包
*.zip
```

注意：`.workbuddy/memory/` 此前被 git 跟踪，已在未提交 diff 中删除，本次提交将正式移除跟踪。

### Step 5: README.md 添加在线 DEMO
